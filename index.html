云计算网站部署实验

云计算环境网站部署实验报告

一、实验基本信息

  • 实验学时：2
  • 实验环境：Windows 11 + Node.js 18.18.0 + Git 2.45.1 + Hexo 4.3.0 + GitHub Pages
  • 实验目标：掌握云计算环境下的网站部署技能，完成个人博客搭建并发布实验报告

二、实验原理

GitHub Pages 是基于GitHub的静态网站托管服务（云计算SaaS层应用），Hexo是轻量级静态博客框架，通过Hexo生成静态页面，再将页面部署到GitHub Pages，实现云计算环境下的个人博客发布。

三、实验步骤

1. 环境搭建

（1）安装Node.js：官网下载稳定版，验证node -v；
（2）安装Git：配置环境变量，验证git --version；
（3）安装Hexo：npm install -g hexo-cli，初始化博客hexo init Hexo-blog。

2. 博客定制

（1）安装Next主题：git clone https://github.com/next-theme/hexo-theme-next.git themes/next；
（2）修改_config.yml：配置博客标题、作者、GitHub Pages地址、启用Next主题；
（3）本地预览：hexo clean && hexo g && hexo s，验证界面定制效果。

3. 部署到GitHub Pages

（1）创建GitHub仓库：Fio9793.github.io；
（2）配置Hexo部署参数：修改deploy项为GitHub仓库地址；
（3）生成个人访问令牌（PAT）：解决GitHub认证问题；
（4）部署命令：hexo clean && hexo g && hexo d，完成部署。

4. 发布实验报告

（1）新建Markdown文章：hexo new "云计算环境网站部署实验报告"；
（2）编写实验报告内容，本地预览格式；
（3）重新部署，将报告发布到博客。

四、实验结果

  1. 环境验证：Node.js/Git/Hexo均安装成功，无版本兼容问题；
  2. 博客定制：成功启用Next主题，界面显示自定义标题、中文语言；
  3. 部署结果：访问https://Fio9793.github.io，能正常显示博客页面；
  4. 报告发布：实验报告已作为博文发布，可在博客中访问。

五、实验问题与解决

  1. 问题1：ssh: connect to host github.com port 22: Connection refused
    解决：改用HTTPS部署方式，配置GitHub个人访问令牌（PAT）替代密码认证。
  2. 问题2：remote: Repository not found（Next主题克隆失败）
    解决：使用最新Next主题仓库地址https://github.com/next-theme/hexo-theme-next.git。
  3. 问题3：LF will be replaced by CRLF警告
    解决：执行git config --global core.autocrlf false消除警告。

六、实验总结

本次实验掌握了Hexo博客框架的使用、GitHub Pages的部署流程，理解了云计算环境下静态网站托管的核心逻辑。通过解决部署过程中的认证、仓库地址、换行符等问题，提升了云计算环境下的故障排查能力，完成了“搭建博客+发布实验报告”的核心目标。
